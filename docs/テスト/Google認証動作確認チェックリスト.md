# Google認証・会員制機能 動作確認チェックリスト

## 基本情報
- **作成日**: 2026年02月15日
- **対象環境**: Localhost (http://localhost:3000)
- **ブラウザ**: Chrome / Edge

---

## 1. ログイン機能 (/auth/signin)
- [ ] **ページ表示**: `/auth/signin` にアクセスし、ログインページが正常に表示されること
- [ ] **UI確認**: 「Googleでログイン」ボタンが表示されていること
- [ ] **Googleログイン動作**:
    - ボタンクリックでGoogleの認証画面（ポップアップまたはリダイレクト）が開くこと
    - 正しいGoogleアカウントでログインできること
- [ ] **リダイレクト**:
    - ログイン成功後、自動的に `/dashboard` （または直前のアクセス元）へリダイレクトされること
    - ログイン済み状態で `/auth/signin` にアクセスした場合、`/dashboard` へリダイレクトされること

## 2. ダッシュボード (/dashboard)
- [ ] **ページ表示**: ログイン後に `/dashboard` が表示されること
- [ ] **ユーザー情報**: ログインしたユーザーの名前やアイコンが表示されていること
- [ ] **保護**: 未ログイン状態で `/dashboard` にアクセスすると、`/auth/signin` へリダイレクトされること

## 3. プロフィール機能 (/profile)
- [ ] **ページ表示**: ヘッダー等のリンクから `/profile` に遷移できること
- [ ] **情報表示**: 登録されているプロフィール情報（名前、メールアドレス等）が表示されていること
- [ ] **編集動作**:
    - プロフィール情報の変更（ニックネーム変更など）ができること
    - 変更後に「保存」ボタンで更新が反映されること
- [ ] **保護**: 未ログイン状態で `/profile` にアクセスすると、`/auth/signin` へリダイレクトされること

## 4. ログアウト機能
- [ ] **ログアウト動作**:
    - メニュー等の「ログアウト」ボタンをクリックしてログアウトできること
    - ログアウト後、トップページまたはログインページに遷移すること
- [ ] **セッション破棄**:
    - ログアウト後にブラウザの「戻る」ボタンでダッシュボードに戻れない（再認証が求められる）こと

## 5. セッション維持・その他
- [ ] **リロード**: ダッシュボード等でページをリロードしてもログイン状態が維持されること
- [ ] **別タブ**: 同じブラウザの別タブで開いてもログイン状態が共有されていること
- [ ] **有効期限**: (仕様にあれば) 一定時間放置後のセッション切れ確認
